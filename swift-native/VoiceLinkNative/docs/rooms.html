VoiceLink Room System

Complete documentation for permanent rooms, guest rooms, device rotation, and capacity management.

Room Types

VoiceLink supports two types of rooms: Guest rooms for unauthenticated users and Permanent rooms for logged-in members.

???
Guest Rooms
Temporary

For users who haven't logged in. Quick and easy but with limitations.

  • Create rooms without login
  • Join some public rooms
  • 10-30 minute duration (random)
  • Maximum 15 members
  • No lock/unlock features
  • 1 room at a time only
  • No password protection
???
Permanent Rooms
Persistent

For logged-in members. Full features with no time limit.

  • Rooms persist indefinitely
  • Lock/unlock rooms
  • Password protection
  • Multiple rooms based on level
  • Higher member capacity
  • Device rotation hosting
  • Full moderation tools
Guest Mode: Rooms expire after a random 10-30 minutes. Sign in to create permanent rooms with full features!

OpenLink Visitor Rooms

When someone uses OpenLink to initiate a voice chat with a visitor (not logged in), a special hidden room is automatically created.

???
OpenLink Visitor Room
Hidden / Auto-Created

These rooms are created automatically when an OpenLink remote connection is established with a visitor.

  • Auto-created by OpenLink system
  • Hidden from public room lists
  • Some VoiceLink features available
  • Not manually creatable by visitors
  • Removed 5 minutes after connection ends
  • Max extension: 10-15 minutes for data collection

OpenLink Room Lifecycle

Phase Duration Description
Active Connection Unlimited Room exists while OpenLink connection is active
Grace Period 5 minutes After connection ends, room stays for cleanup
Extended (if needed) +5-10 minutes Extra time for data collection in some cases
Maximum Total 15 minutes post-connection Room is forcefully removed after this
Visitor vs Guest: Visitors cannot create rooms themselves. OpenLink creates rooms on their behalf when initiating a remote connection. Guests (users who visited but haven't logged in) can create temporary rooms.

Device Rotation

When you have multiple devices linked, rooms can be hosted across different devices using rotation modes.

???
Random

Randomly select a device for each room

???
Round Robin

Rotate through devices in order

???
Preferred

Use your preferred device when available

???
Load Balanced

Distribute based on device capacity

How Device Rotation Works

  • All devices stay in sync with room state
  • Rooms automatically migrate if host device goes offline
  • You can set a preferred host device
  • Load balancing considers current room assignments
