VoiceLink TestFlight - What to Test
Build 79 / 1.0.0
Date: 2026-06-02

Before testing, open TestFlight and update to the latest VoiceLink build.

Focus for this build: confirm the emergency launch-crash fixes, safer Servers tab behavior, account sign-in state, Mastodon sign-in, server support tickets, room join stability, room system messages, audio controls, scrolling/audio performance, and accessibility. Recent tester reports were mostly older-build launch crashes, server-list crashes, audio buffering, Mastodon sign-in failure, and contact-form validation errors, so please retest those areas first.

1. Launch and Servers tab
- Launch the app and confirm it reaches the native UI without crashing.
- Open Servers. It should default to the safer Federated Rooms view and remain stable when refreshed.
- Confirm the four expected server rows are uniquely named for VoiceLink Main, VoiceLink Community, DevineCreations on devine-creations.com, and DevineCreations on devinecreations.net.
- If any extra generic Community row appears, report its URL.

2. Sign-in
- Sign in with a VoiceLink account from Settings.
- Confirm sign-in fields disappear after success and the signed-in summary plus Sign Out action appear.
- Test username or email login if both are available.
- Try Mastodon sign-in with a suggested instance and a custom instance. Layer8 should work with layer8.space or layer8.

3. Rooms and support tickets
- Join rooms from Main and Community as guest and signed-in user.
- Confirm presence, messages, join/leave system messages, and first-join server terms behave correctly.
- From server details or room details, create a short support ticket. Confirm server/room context is included and ticket details do not post into the room.

4. VoiceOver and accessibility
- With VoiceOver on, focus a person in a room. Double tap should open that profile.
- Confirm View Profile, Direct Message, monitor, whisper, and audio-control actions remain reachable where expected.
- Server rows, room rows, users, controls, status states, and missing descriptions should have useful spoken labels.

5. Audio and performance
- Confirm microphone audio reaches other room clients and incoming room audio plays on iOS.
- Open and close audio controls while staying joined.
- Scroll room and message lists while audio is active. Report buffering, heat, stutter, or stuck loading.
- Background and foreground the app, then rejoin a room.

Please report device model, iOS version, build number, server used, exact steps, expected result, actual result, and a screenshot or recording when possible.
