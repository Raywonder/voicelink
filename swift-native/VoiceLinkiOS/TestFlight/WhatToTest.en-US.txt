Build 91 focuses on owner-button server browsing.

What changed:
- Server owners now appear as buttons instead of always-open headings.
- Each owner button shows how many servers that owner runs, such as VoiceLink (2 servers) or Devine Creations (2 servers).
- Opening an owner button reveals that owner's servers. This keeps each platform owner's server set together and avoids mixing all servers into one long list.
- Searching on iOS still exposes matching owner groups so search results are not hidden.
- User, administrator, and governance docs now describe owner buttons and server-count behavior.

Please test:
- Open Servers on iOS and confirm VoiceLink and Devine Creations appear as owner buttons with server counts.
- Open VoiceLink and confirm its servers appear only after opening that owner button.
- Open Devine Creations and confirm its servers appear only after opening that owner button.
- On macOS, open the Servers tab and confirm the same owner-button behavior.
- Confirm selecting a server still switches to browsing that server's rooms.
- Confirm existing room browsing, preview, join, user list, messages, and microphone status still work after the server browser change.
