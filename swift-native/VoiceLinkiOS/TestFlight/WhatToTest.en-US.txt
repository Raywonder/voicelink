VoiceLink TestFlight - What to Test

Build focus: audio engine upgrade test pass.

This build starts moving VoiceLink toward the new professional audio pipeline. It does not claim the final Miniaudio plus Opus replacement is complete yet, but it changes the client and server defaults so testers can verify Original Audio behavior before the deeper codec engine lands.

Please test:

- Join a room on iPhone and confirm microphone audio still transmits.
- Confirm room audio is still received from another participant.
- Confirm Original Audio is the default audio mode.
- Confirm noise suppression, echo cancellation, and auto gain are not forced on by default.
- In room audio controls, switch between Original Audio, Voice Isolation, Meeting Mode, and Studio Mode.
- Confirm audio mode changes do not require leaving and rejoining the room.
- Test with built-in speaker, wired headphones, Bluetooth headset, and AirPods if available.
- Test with the screen locked for at least two minutes while staying in a room.
- Test after changing audio route while connected.
- On Mac, test room audio with the matching macOS build from this branch.

Known limitations for this build:

- Opus encode/decode is advertised as the preferred future codec, but the current relay still falls back to Float32 PCM packets.
- Miniaudio is represented by the new server/client capability module and defaults, but the full native Miniaudio device layer is not fully linked yet.
- PA/whisper-specific paths may still use speech processing intentionally.

Report any distortion, clipping, one-sided stereo, device switching failures, or audio mode changes that do not take effect.
