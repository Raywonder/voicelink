VoiceLink Bug Tracker

Release baseline: VoiceLink 1.0.0 Build 26

Last updated: 2026-03-04

Current Public Status

  • Desktop installers are the active supported release path.
  • Main and community VPS downloads are mirrored on the current build 26 baseline.
  • Public browser room access is intentionally closed until browser readiness is complete.

Current Focus

  1. Desktop media playback consistency for room preview and in-room stream startup.
  2. Licensing and device activation flow clarity for authenticated desktop users.
  3. Profile, account-linking, and provider-aware account management in desktop settings.
  4. Messaging, direct messaging, attachment handling, and bot interaction polish.
  5. Server administration coverage, module configuration, and advanced settings persistence.
  6. Background streams, metadata, and room-level media controls.
  7. Startup behavior, room restore, and window/minimize behavior consistency.
  8. Mastodon/server watchdog and service-recovery automation.

Release Notes Context

Use this page as the current issue and readiness summary for the desktop-first VoiceLink baseline. Older January and February issue lists are superseded by the current build 26 release state.

Report Path

If a problem is reproducible on the current build, report it against the build 26 desktop baseline and include platform, auth method, and room/server context.
