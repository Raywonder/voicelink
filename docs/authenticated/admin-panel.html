VoiceLink Admin Guide

Admin features are role-based. Only authorized roles can edit server settings.

Desktop Admin Access

  • Desktop app refreshes admin role on login and server reconnect.
  • Server Administration menu appears only when admin role is active.

Why you would use this: this is the fastest way to change room policy, fix server settings, or verify what members are seeing without logging into the server directly.

Example: if a room is locked by mistake, open Server Administration from the desktop app, review who locked it, and unlock it without leaving the room browser.

Moderation Controls

  • User moderation: kick, ban, mute policy, room access enforcement.
  • Room moderation: lock, privacy behavior, and preview restrictions.

Example: lock a room when a staff discussion starts, leave current members inside, and reopen it later when you are ready for new joins again.

Audio and Policy

  • Publish baseline settings for noise suppression, echo cancellation, and gain control.
  • Require room-only sounds for join/leave and avoid autoplay UI noise on page load.

Update Operations

  • Keep macOS package filename stable as VoiceLinkMacOS.zip.
  • Refresh checksum on every new build, even when version remains 1.0.
  • Fallback to IP endpoints if domain fails, then auto-return to domain when healthy.

Example: if you publish a hotfix build the same day, update the package, checksum, and metadata together so desktop auto-update does not offer stale downloads.

Log Collection

  • Enable Send Logs in Advanced settings for support diagnostics.
  • Collect app version, endpoint used, and room join/audio errors.

Example: use this after a bug report that says audio dropped, a room vanished, or a server switch failed. The logs make it easier to see whether the problem was auth, media, federation, or room state.
