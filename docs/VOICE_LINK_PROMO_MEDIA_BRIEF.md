# VoiceLink Promo Media Brief

Last updated: 2026-03-26

## Goal

Create one accessible promotional video for VoiceLink that feels cinematic and modern, but remains grounded in real product behavior.

The video should work with:

- spoken description track
- captions
- clean still-image derivatives for download pages and bug trackers

## Video Direction

- Format: one combined launch video
- Tone: confident, welcoming, social, accessibility-first
- Style reference: polished AI-assisted product reel in the spirit of Sora-style motion, but based on real VoiceLink features
- Runtime target: 45 to 75 seconds

## Must Show

1. Room discovery and quick join
2. Members list and chat in-room
3. Spatial / room audio feel
4. Live transcripts panel
5. Admin controls and room management
6. VoiceOver / accessibility cues
7. Desktop, web, and iOS presence in one product story

## Accessibility Requirements

- Captions burned in or delivered as a paired track
- Full transcript published with the video
- Spoken audio description version or integrated described narration
- No critical information shown by animation alone
- Any still images exported from the video need plain-language alt text

## Suggested Sequence

1. Open on VoiceLink logo and a calm room ambience bed.
2. Show desktop room list and a room join.
3. Cut to full sender names in chat and members list activity.
4. Show admin opening room controls and server settings.
5. Show live transcript lines appearing while someone speaks.
6. Show web access and iOS connection as companion clients.
7. Close on the product family:
   - VoiceLink
   - VoiceLink Web Access
   - TestFlight on iOS

## Still Image Set

- Hero desktop screenshot with room list and active room
- Web login / guest access screenshot
- Admin settings screenshot
- Live transcripts screenshot
- Room media / ambience screenshot

Each still should ship with:

- title
- one-sentence description
- alt text

## Delivery Notes

- This file is a content brief, not a generated video asset.
- If Sora or another video generator is used later, keep the output anchored to real current UI and current feature wording.
