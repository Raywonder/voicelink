VoiceLink Public User Guide

This guide is for all users on web and desktop clients.

Quick Start

  • Open the room list, pick a room, and select Join.
  • Use room context menu for Preview Room when allowed.
  • If preview is blocked, VoiceLink shows a privacy message.

Audio Basics

  • Desktop: use Audio menu for mute/deafen, device selection, and processing toggles.
  • Safari iOS: confirm Microphone permission in Safari site settings and reload once.
  • If room audio fails, leave and rejoin room after confirming device route.

Notifications

  • Join/leave notifications can be toggled in Notifications settings.
  • Room preview privacy messages are fixed and not user-editable.

File Sharing

  • Receive mode can be set to auto, ask always, ask once, or block.
  • Admins can enforce file policy at server level.

Update Behavior

  • Desktop updater checks domain endpoints first.
  • If domain fails, fallback API endpoints may use IP automatically.
  • When domain is healthy again, updater returns to domain endpoints.
  • Windows desktop installer links are intentionally hidden for now; macOS is the active desktop build.
