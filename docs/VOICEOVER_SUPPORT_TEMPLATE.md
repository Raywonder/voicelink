# Using VoiceLink with VoiceOver

This template is the standard format for VoiceOver-focused support pages in VoiceLink. Use it for in-app help, website support articles, and mirrored documentation bundles.

## Intro

VoiceLink is intended to work well with VoiceOver. This guide format covers the main tasks, navigation patterns, and controls a screen-reader user needs without extra marketing or implementation detail.

## Basics

Explain the main areas of the app in plain language.

Include:
- what the primary views are called
- how focus moves between them
- what happens when the app first opens

## Getting Started

Explain the first successful task.

Examples:
- open the app
- choose a server
- open a room
- join as a guest or sign in
- confirm the room opened successfully

## Main Navigation

Document how users move between major areas.

Include:
- tabs
- room lists
- settings
- profile or messaging views
- dialogs and sheets

## Working with Rooms

Split by task:
- previewing a room
- opening room details
- joining a room
- reviewing people in the room
- reading room messages and system messages
- adjusting room audio controls

## VoiceOver and Keyboard Notes

List only practical commands and behaviors.

Examples:
- VO-Space activates the selected item
- VoiceOver actions on a room row can expose Preview, Join, or Share
- sliders should announce percentages and change in consistent steps
- Escape closes dialogs where supported

## Troubleshooting

Keep this short and direct.

Examples:
- if no room users are shown, refresh the room and send diagnostics
- if no audio is heard, check output mute and selected output
- if a secure connection fails, the app may retry the fallback transport when allowed by settings

## Contact and Feedback

End with one short paragraph explaining how to contact support or send diagnostics.

## Writing Rules

- Use short paragraphs.
- Use flat bullets only.
- Use exact UI labels where possible.
- Avoid implementation details.
- Avoid marketing language.
- Keep support text understandable through VoiceOver in one pass.
